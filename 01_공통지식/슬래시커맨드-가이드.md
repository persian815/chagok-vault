
> Claude Code CLI에서 쓰는 `/skills-*` 커맨드 목록과 사용 시점.
> 파일 위치: `CHAGOK/.claude/commands/`

---

## 마무리 루틴

세션 끝낼 때 항상 이 순서로 실행한다.

```
/skills-end
```

내부적으로 아래 4단계를 순서대로 실행:

| 순서 | 커맨드 | 역할 |
|---|---|---|
| 1 | `/skills-cr` | 코드 리뷰 — 보안·패턴·버그 검토 및 즉시 수정 |
| 2 | `/skills-build` | 빌드 체크 — TypeScript·프로덕션 빌드 통과 확인 |
| 3 | `/skills-docs` | 문서 동기화 — CLAUDE.md 파일들 최신화 |
| 4 | `/skills-push` | 커밋 & 푸시 — origin/main에 반영 |

## 새 프로젝트

| 커맨드 | 역할 | 실행 시점 |
|---|---|---|
| `/skills-design` | 요구사항 → 구조화된 프로젝트 설계 문서 생성 | `02_기획/요구사항/` 파일 작성 후 |
| `/skills-project [이름]` | 설계 문서 → vault md 5개 + CLAUDE.md 생성 | `/skills-design` 실행 후 |
| `/skills-init [이름]` | CLAUDE.md → Next.js 스캐폴딩 + 패키지 설치 | `/skills-project` 실행 후 |

**사용 흐름:**
```
1. vault/02_기획/요구사항/YYYY-MM-DD-요청_[이름].md 작성
       ↓
2. /skills-design 실행
   → vault/02_기획/설계/[이름]_설계.md 생성
     (기술스택·구조·DB·ADR·기능목록 포함)
       ↓
3. /skills-project [이름] 실행
   → vault/03_프로젝트/[이름]/ 00~04.md (옵시디언 기록용)
   → projects/[이름]/CLAUDE.md (바이브코딩 세션용)
   → 브랜드가이드·로드맵 업데이트
       ↓
4. /skills-init [이름] 실행
   → Next.js 초기화 + 패키지 설치
   → 폴더·컴포넌트·API 스텁 생성
   → .env.local 템플릿 + supabase/migrations/ 생성
   → 바이브코딩 시작 준비 완료
```

## vault 동기화

| 커맨드 | 역할 | 실행 시점 |
|---|---|---|
| `/skills-vault` | `projects/*/CLAUDE.md` → vault `00~02.md` + 로드맵 최신화 | 개발 세션 후, 버전 올릴 때 |

업데이트 대상:
- `vault/03_프로젝트/[이름]/00_개요.md` — 버전·상태·인프라 현황
- `vault/03_프로젝트/[이름]/01_설계.md` — 프로젝트 구조·DB 테이블
- `vault/03_프로젝트/[이름]/02_기능목록.md` — 기능 완료 여부
- `vault/00_브랜드/프로젝트-로드맵.md` — 프로젝트 상태 트리·다음 계획

## 옵시디언 → Claude Code 연결 흐름

```
1. 옵시디언에서 오늘 할 것 확인
       ↓
2. Claude Code 실행 (CLAUDE.md 자동 로딩)
       ↓
3. 개발 중 중요한 결정·삽질 → 옵시디언에 기록
       ↓
4. 세션 마무리: /skills-end
```

## 커맨드 추가하는 법

새 커맨드가 필요하면:
1. `CHAGOK/.claude/commands/skills-[이름].md` 파일 생성
2. 실행 절차를 단계별로 작성
3. 이 가이드 문서에 추가
