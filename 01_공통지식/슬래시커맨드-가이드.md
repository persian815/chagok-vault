
> Claude Code CLI에서 쓰는 `/skills-*` 커맨드 목록과 사용 시점.
> 파일 위치: `littleRecords/.claude/commands/`

---

## 마무리 루틴

세션 끝낼 때 항상 이 순서로 실행한다.

```
/skills-end
```

내부적으로 아래 4단계를 순서대로 실행:

| 순서 | 커맨드 | 역할 |
|---|---|---|
| 1 | `/skills-cr` | 코드 리뷰 — 보안·패턴·버그 검토 및 즉시 수정 |
| 2 | `/skills-build` | 빌드 체크 — TypeScript·프로덕션 빌드 통과 확인 |
| 3 | `/skills-docs` | 문서 동기화 — CLAUDE.md 파일들 최신화 |
| 4 | `/skills-push` | 커밋 & 푸시 — origin/main에 반영 |

---

## 블로그 발행

| 커맨드 | 역할 | 실행 시점 |
|---|---|---|
| `/skills-blog` | 오늘 작업 내용 → 블로그 글 자동 작성 후 발행 | 작업 마무리 후 |
| `/skills-claude-post` | 클로드의 코딩 일기 작성 후 발행 | 하루 마무리 후 |
| `/skills-posts` | 옵시디언에서 쓴 .md 파일 즉시 발행 | 글 바로 올리고 싶을 때 |

**발행 경로:** `blog/posts/*.md` → `persian815/blog` 레포 push → 알뜰살뜰 앱 블로그 탭에서 바로 보임

## 기타

| 커맨드 | 역할 | 실행 시점 |
|---|---|---|
| `/skills-request` | `blog/request/*.md` 읽고 답변 파일 생성 | 옵시디언에서 질문 작성 후 |

---

## 옵시디언 → Claude Code 연결 흐름

```
1. 옵시디언에서 오늘 할 것 확인
       ↓
2. Claude Code 실행 (CLAUDE.md 자동 로딩)
       ↓
3. 개발 중 중요한 결정·삽질 → 옵시디언에 기록
       ↓
4. 세션 마무리: /skills-end
       ↓
5. 블로그 발행: /skills-blog 또는 /skills-claude-post
```

---

## 커맨드 추가하는 법

새 커맨드가 필요하면:
1. `littleRecords/.claude/commands/skills-[이름].md` 파일 생성
2. 실행 절차를 단계별로 작성
3. 이 가이드 문서에 추가
