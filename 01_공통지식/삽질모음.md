## Vercel

### 배포가 너무 자주 트리거됨
- **원인:** 코드 수정할 때마다 GitHub push → 자동 배포 반복
- **해결:** 작업 중엔 commit만, 마무리할 때 한 번만 push
- **출처:** 알뜰살뜰 배포 첫날 (45분에 12번 배포)

### 이미지 업로드 500 에러
- **원인:** Vercel 기본 요청 크기 제한 4.5MB. 스마트폰 캡처 이미지가 초과
- **해결:** `next.config.mjs`에 `bodySizeLimit: '10mb'` 설정
- **출처:** 알뜰살뜰 배포 당일

### MIDDLEWARE_INVOCATION_FAILED
- **원인:** 환경변수 없을 때 `createServerClient`가 크래시
- **해결:** 미들웨어에 null 체크 추가
- **출처:** 알뜰살뜰 배포 당일

## Cerebras

### 모델명이 문서와 다름
- **원인:** 공식 문서에 있는 모델명이 실제로는 존재하지 않는 경우 있음
- **시도한 것들:** `qwen-3-235b-a22b-instruct-2507` → 400, `llama-3.3-70b` → 404, `llama3.3-70b` → 404
- **해결:** API에 직접 찔러서 확인. 현재 안정적인 모델: `llama3.1-8b`
- **교훈:** Cerebras는 문서보다 API 직접 확인이 빠름

### `thinking` 파라미터 400 에러
- **원인:** Cerebras는 thinking 파라미터 미지원
- **해결:** 파라미터 제거
- **출처:** 알뜰살뜰 개발 중

### 토큰 한도 초과 (context_length_exceeded)
- **원인:** 전체 구매내역을 AI에 넣으면 8192 토큰 초과
- **해결:** 최근 3개월 + 150건으로 제한
- **출처:** 알뜰살뜰 배포 당일

## Gemini

### JSON 파싱 실패
- **원인:** `maxOutputTokens: 2048`이 너무 작아서 응답이 중간에 잘림
- **해결:** 8192로 올리고, `[` ~ `]` 구간만 추출하는 파싱 로직 추가
- **출처:** 알뜰살뜰 개발 중

## Supabase

### 게스트 로그인 실패 (anonymous_provider_disabled)
- **원인:** Supabase 대시보드에서 Anonymous 인증을 켜지 않음
- **해결:** Authentication → Providers → Anonymous 토글 활성화
- **교훈:** 에러 메시지를 숨기지 말고 원문 그대로 보여줄 것

### RLS로 인해 logAI 실패
- **원인:** `logAI` 내부에서 `createClient()` 재생성 시 응답 후 컨텍스트 소멸
- **해결:** 이미 인증된 supabase 인스턴스를 인자로 넘길 것
  ```typescript
  // ❌ logAI 내부에서 createClient() 재생성
  // ✅ logAI({ supabase, userId, ... }) 형태로 인스턴스 전달
  ```
- **출처:** 알뜰살뜰 개발 중
