---

## 기본 원칙

차곡 프로젝트는 대부분 **소규모 (1~5인)** 앱이다.

- 복잡한 인프라 불필요
- 무료 플랜으로 최대한 버티기
- 모바일에서 잘 돌아야 함
- 배포는 자동화 (GitHub push → 끝)

---

## 검증된 스택 (알뜰살뜰 기준)

| 분류       | 선택                      | 이유                         | 주의사항                                 |
| -------- | ----------------------- | -------------------------- | ------------------------------------ |
| Frontend | Next.js 14 + TypeScript | App Router로 서버/클라이언트 분리 깔끔 | strict mode 필수                       |
| Styling  | Tailwind + shadcn/ui    | 모바일 UI 빠르게 가능              | `@tailwindcss/typography` 따로 설치 필요   |
| DB/Auth  | Supabase                | RLS로 인증 붙이기 쉬움, 무료 플랜 충분   | Anonymous Auth는 대시보드에서 직접 활성화        |
| 이미지 AI   | Gemini 2.5 Flash        | 이미지 분석 성능·비용 균형 좋음         | maxOutputTokens 8192 이상 설정           |
| 텍스트 AI   | Cerebras                | 완전 무료, 빠름                  | 모델명 반드시 API 직접 확인. 현재: `llama3.1-8b` |
| 배포       | Vercel                  | GitHub push만 하면 자동 배포      | 작업 중엔 push 자제, 마무리 때 한 번만            |

## 결정 기준 체크리스트

새 프로젝트 기술스택 결정 전 확인:

- [ ] 사용자 수는? (1~5인 / 소규모 / 대규모)
- [ ] 이미지 처리 필요한가? → Gemini
- [ ] 텍스트 AI 필요한가? → Cerebras (무료) or Gemini
- [ ] 실시간 기능 필요한가? → Supabase Realtime 고려
- [ ] 모바일 전용인가? → PWA or React Native 고려
- [ ] 무료 플랜으로 버틸 수 있나?
