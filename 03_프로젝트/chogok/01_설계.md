## DB 스키마

### profiles
```sql
CREATE TABLE profiles (
  id         uuid PRIMARY KEY REFERENCES auth.users ON DELETE CASCADE,
  name       text,
  created_at timestamptz DEFAULT now()
);
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON profiles FOR ALL USING (auth.uid() = id);
```

### episodes
```sql
CREATE TABLE episodes (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES auth.users NOT NULL,
  ep          integer NOT NULL,
  date        date NOT NULL,
  area        text,
  place       text,
  trip        text,
  memo        text,
  lat         numeric,
  lng         numeric,
  photos      text[],
  created_at  timestamptz DEFAULT now()
);
ALTER TABLE episodes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON episodes FOR ALL USING (auth.uid() = user_id);
CREATE UNIQUE INDEX episodes_user_ep_idx ON episodes (user_id, ep);
```

### purchases (spending)
```sql
CREATE TABLE purchases (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES auth.users NOT NULL,
  date        date NOT NULL,
  shop        text NOT NULL,
  item        text NOT NULL,
  price       integer NOT NULL,
  discount    integer DEFAULT 0,
  category    text,
  image_url   text,
  created_at  timestamptz DEFAULT now()
);
ALTER TABLE purchases ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON purchases FOR ALL USING (auth.uid() = user_id);
```

### projects
```sql
CREATE TABLE projects (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid REFERENCES auth.users NOT NULL,
  name        text NOT NULL,
  icon        text DEFAULT '📋',
  due_date    date,
  sort_order  integer DEFAULT 0,
  created_at  timestamptz DEFAULT now()
);
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON projects FOR ALL USING (auth.uid() = user_id);
```

### tasks
```sql
CREATE TABLE tasks (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id   uuid REFERENCES projects(id) ON DELETE CASCADE NOT NULL,
  user_id      uuid REFERENCES auth.users NOT NULL,
  title        text NOT NULL,
  month        text,
  priority     text CHECK (priority IN ('최우선','중요','진행')),
  assignee     text,
  detail       text,
  completed    boolean DEFAULT false,
  completed_at timestamptz,
  sort_order   integer DEFAULT 0,
  created_at   timestamptz DEFAULT now()
);
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON tasks FOR ALL USING (auth.uid() = user_id);
```

### events
```sql
CREATE TABLE events (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id  uuid REFERENCES projects(id) ON DELETE CASCADE NOT NULL,

ALTER TABLE events ENABLE ROW LEVEL SECURITY;

```

## 핵심 설계 결정 (ADR)

### ADR-001: 통합 앱 vs 기존 앱 유지
- **결정:** 세 앱을 chogok 하나로 통합
- **이유:** 홈 캘린더가 세 도메인 데이터를 동시에 보여야 하므로 분리된 앱으로는 구현 불가
- **구현:** 기존 Supabase 데이터 마이그레이션 또는 새 프로젝트로 시작

### ADR-002: couples 테이블 제거 → user_id 기반 격리
- **결정:** 두리의 `couples` 테이블 구조를 버리고 `user_id` 기반 단순 격리
- **이유:** 부부 공유를 위해 같은 계정 사용 또는 각자 계정으로 접속. 커플 연결 로직 불필요
- **구현:** 모든 테이블 `user_id` 컬럼 + RLS

### ADR-003: 할일을 "프로젝트" 개념으로 확장
- **결정:** tasks가 `project_id`에 속하는 구조. 결혼 준비는 projects 중 하나
- **이유:** 결혼 후에도 신혼집 꾸미기, 여행 등 계속 사용 가능
- **구현:** 첫 로그인 시 "결혼 준비" 프로젝트 seed 자동 생성

### ADR-004: PWA (next-pwa)
- **결정:** React Native 대신 Next.js PWA
- **이유:** 기존 Next.js 스택 재활용, 웹 배포로 양쪽 다 접근 가능
- **구현:** `next-pwa` 패키지 + `manifest.json` + 아이콘 2종 (192, 512)
- **주의:** Tailwind 4는 v3과 설정 방식이 다름, next-pwa의 Next.js 15 호환성 확인 필요

### ADR-005: 통합 캘린더 API
- **결정:** `/api/home/calendar` 단일 엔드포인트에서 3개 테이블 병렬 쿼리
- **이유:** 캘린더 렌더링 시 왕복 3번보다 1번이 성능상 유리
- **구현:**
  ```typescript
  const [episodes, events, purchases] = await Promise.all([
    supabase.from('episodes').select(...).gte('date', startDate).lte('date', endDate),
    supabase.from('events').select(...).gte('date', startDate).lte('date', endDate),
    supabase.from('purchases').select(...).gte('date', startDate).lte('date', endDate),
  ])
  ```

### ADR-006: 색상 팔레트 CSS 변수
- **결정:** 도메인별 색상을 CSS 변수로 관리
- **구현:**
  ```css
  :root {
    --brand:    #3D6B4F; /* 세이지 그린 — 브랜드, 오늘 날짜 */
    --episode:  #D86344; /* 코랄 — 에피소드 dot */
    --todo:     #B5825A; /* 로즈골드 — 할일 dot */
    --spending: #3D7D5A; /* 세이지 그린(진) — 소비 dot */
  }
  ```
