## DB 스키마

| 테이블 | 용도 |
|---|---|
| `episodes` | 에피소드 기록 (ep, date, area, place, trip, start_time, end_time, memo, location, lat, lng, photos) |

### episodes

```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES auth.users NOT NULL
ep          integer                           -- 에피소드 번호 (unique per user)
date        date NOT NULL
area        text                              -- 지역 (예: 서울)
place       text                              -- 장소명
trip        text                              -- 여행명
start_time  time
end_time    time
memo        text
location    text                              -- 주소 (geocode 결과)
lat         numeric
lng         numeric
photos      text[]                            -- Storage 경로 배열
created_at  timestamptz DEFAULT now()
```

RLS 활성화: `auth.uid() = user_id` 정책으로 본인 데이터만 접근.
인덱스: `episodes_user_id_ep_idx` (unique)

## 핵심 설계 결정 (ADR)

### ADR-001: 에피소드 vs 활동 구분
- **결정:** `isEpisode` 토글로 두 모드 분기 — 에피소드는 Supabase 저장, 활동은 ICS만 생성
- **이유:** 날짜별 중요 기억(에피소드)과 단발성 활동을 분리해 DB 과적재 방지
- **날짜:** 2026-06-01

### ADR-002: EXIF 자동 파싱
- **결정:** PhotoPicker에서 사진 선택 시 EXIF로 날짜·GPS 자동 입력, 수동 수정 가능
- **이유:** 사진에서 날짜·장소 직접 입력하는 마찰 제거
- **구현:** `lib/exif.ts` → `lib/geocode.ts` (GPS → 주소)
- **날짜:** 2026-06-01

### ADR-003: ICS 출력으로 캘린더 연동
- **결정:** 에피소드 저장 시 ICS 파일 자동 다운로드, 히스토리에서 재다운로드 가능
- **이유:** Apple Calendar·Google Calendar 등 기존 캘린더 앱과 연동
- **날짜:** 2026-06-01

### ADR-004: proxy.ts 인증 게이트웨이
- **결정:** Next.js middleware 대신 `proxy.ts`에서 인증 체크 후 리다이렉트
- **이유:** 전체 앱을 로그인 없이 접근 불가 처리
- **날짜:** 2026-06-01
