## DB 스키마

### wedding_guestbook

```sql
CREATE TABLE wedding_guestbook (
  id         uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name       text NOT NULL,
  message    text NOT NULL,
  created_at timestamptz DEFAULT now()
);
-- RLS 없이 공개: 누구나 읽기·쓰기 가능
-- 삭제는 service_role key로만 (관리자)
```

### wedding_attendance

```sql
CREATE TABLE wedding_attendance (
  id         uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name       text NOT NULL,
  side       text NOT NULL,       -- '신랑' | '신부'
  attending  text NOT NULL,       -- '참석' | '불참석'
  meal       text NOT NULL,       -- '○' | '×' | '미정'
  companion  text,                -- 동반인 이름
  message    text,                -- 전달 메시지
  created_at timestamptz DEFAULT now()
);
```

### wedding_config

```sql
CREATE TABLE wedding_config (
  id         integer PRIMARY KEY,  -- 항상 1 (싱글 레코드)
  data       jsonb NOT NULL,       -- WeddingConfig JSON
  updated_at timestamptz
);
-- WeddingConfig: invitation, venue, people, interview, ourStory,
--               infoTabs, accounts, attendanceFormUrl,
--               ceremonyImage, bgmUrl, coverImage, gallery
```

## 핵심 설계 결정 (ADR)

### ADR-001: 방명록 인증 없는 공개 접근
- **결정:** RLS 비활성화, 누구나 읽기·쓰기 가능
- **이유:** 하객이 별도 로그인 없이 방명록 작성 가능해야 함
- **삭제 보호:** `x-admin-password` 헤더 + service_role key로만 DELETE 가능
- **날짜:** 2026-06-16

### ADR-002: 배경음악 팝업 닫기 시 재생
- **결정:** 페이지 진입 시 자동재생 대신, 진입 팝업 닫기 버튼 클릭 시 재생 시작
- **이유:** 브라우저 자동재생 정책(사용자 인터랙션 없는 오디오 차단)을 우회
- **날짜:** 2026-06-16

### ADR-003: 게스트스냅 혼례일 이후만 활성
- **결정:** 혼례일(2026-09-19) 이전에는 업로드 UI 비활성화, 앱에서 날짜 체크
- **이유:** 혼례 당일 현장에서만 스냅 공유하는 것이 자연스러운 UX
- **구현:** `new Date() >= new Date('2026-09-19')` 체크 후 UI 분기
- **날짜:** 2026-06-16

### ADR-004: 달력 외부 라이브러리 미사용
- **결정:** 9월 달력을 정적으로 직접 렌더링, 19일 셀에만 골드 원 스타일
- **이유:** 라이브러리 bundle size 절감, 스타일 커스텀 자유도
- **날짜:** 2026-06-16

### ADR-005: 관리자 페이지 단순 비밀번호 보호
- **결정:** Next Auth 등 인증 시스템 없이 `ADMIN_PASSWORD` 환경변수 단순 비교
- **이유:** 단일 관리자(신랑/신부), 별도 계정 관리 불필요. sessionStorage에 임시 저장
- **날짜:** 2026-06-16

### ADR-006: WeddingConfig DB 저장 (환경변수 대신)
- **결정:** 청첩장 콘텐츠(초대 문구, 예식장, 계좌, Q&A, 스토리 등)를 `wedding_config` 테이블 JSON으로 저장
- **이유:** 코드 재배포 없이 관리자 UI에서 콘텐츠 수정 가능. 이미지 URL도 동일 JSON에 포함
- **날짜:** 2026-06-16

### ADR-007: 참석여부 자체 DB 수집
- **결정:** 구글폼 외부 링크 대신 `wedding_attendance` 테이블에 직접 수집
- **이유:** 인앱 UX 유지, 데이터를 직접 관리 가능. 관리자 페이지에서 조회 가능
- **날짜:** 2026-06-16

### ADR-008: BGM YouTube URL 기반
- **결정:** `/public/audio/bgm.mp3` 파일 대신 YouTube URL을 admin에서 설정
- **이유:** 파일 용량 문제 없음, 곡 변경 시 재배포 불필요. BgmContext로 컴포넌트 간 상태 공유
- **날짜:** 2026-06-16

### ADR-009: 지도 Google Maps iframe + 네이버 버튼
- **결정:** 주소 기반 Google Maps를 iframe으로 임베드, 네이버 지도 장소 ID는 외부 링크 버튼으로 제공
- **이유:** 네이버 지도 iframe은 도메인 등록 필요, Google Maps가 더 간단
- **날짜:** 2026-06-16
