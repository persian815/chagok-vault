## DB 스키마

### wedding_guestbook

```sql
CREATE TABLE wedding_guestbook (
  id         uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name       text NOT NULL,
  message    text NOT NULL,
  created_at timestamptz DEFAULT now()
);
-- RLS 없이 공개: 누구나 읽기·쓰기 가능
-- 삭제는 service_role key로만 (관리자)
```

## 핵심 설계 결정 (ADR)

### ADR-001: 방명록 인증 없는 공개 접근
- **결정:** RLS 비활성화, 누구나 읽기·쓰기 가능
- **이유:** 하객이 별도 로그인 없이 방명록 작성 가능해야 함
- **삭제 보호:** `SUPABASE_SERVICE_ROLE_KEY`로만 DELETE 가능 (API 라우트에서 서버 사이드 처리)
- **날짜:** 2026-06-16

### ADR-002: 배경음악 팝업 닫기 시 재생
- **결정:** 페이지 진입 시 자동재생 대신, 진입 팝업 닫기 버튼 클릭 시 `audioRef.current?.play()` 호출
- **이유:** 브라우저 자동재생 정책(사용자 인터랙션 없는 오디오 차단)을 우회
- **날짜:** 2026-06-16

### ADR-003: 게스트스냅 혼례일 이후만 활성
- **결정:** 혼례일(2026-09-19) 이전에는 업로드 UI 비활성화, 앱에서 날짜 체크
- **이유:** 혼례 당일 현장에서만 스냅 공유하는 것이 자연스러운 UX
- **구현:** `new Date() >= new Date('2026-09-19')` 체크 후 UI 분기
- **날짜:** 2026-06-16

### ADR-004: 달력 외부 라이브러리 미사용
- **결정:** 9월 달력을 정적으로 직접 렌더링, 19일 셀에만 골드 원 스타일
- **이유:** 라이브러리 bundle size 절감, 스타일 커스텀 자유도
- **날짜:** 2026-06-16

### ADR-005: 참석여부 구글폼 외부 연결
- **결정:** 참석여부 수집은 구글 폼으로 외부 연결
- **이유:** 별도 DB·관리 UI 불필요. 구글 폼이 응답 집계·알림 등 기능 충분
- **날짜:** 2026-06-16
