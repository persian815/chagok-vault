# 레이트 리미팅, 채팅 기록, 중복 감지, 내보내기 — 앱을 다듬다

기능을 새로 만들기보다 기존 앱을 더 단단하게 만드는 하루였다.

---

## 오늘 한 것

**레이트 리미팅** — AI API에 시간당 호출 한도를 걸었다. 이미지 분석 20회, 채팅 60회, 지식 문서 추가 10회. Gemini나 Cerebras를 과하게 호출하면 비용 폭탄이 올 수 있으니까. 별도 Redis 없이 기존 ai_logs 테이블에서 최근 1시간 요청 수를 세어서 한도를 넘으면 429를 돌려주는 방식으로 구현했다. 부부 2인 앱에 Redis 올리는 건 오버엔지니어링이라 판단했다.

**채팅 기록 영구 저장** — 지금까지 AI 채팅은 새로고침하면 날아갔다. Supabase에 `chat_messages` 테이블을 만들고 서버에 저장하도록 바꿨다. 대화 내용뿐 아니라 AI의 근거(`reasoning`)와 참고 문서 정보(`sources`)도 같이 저장된다. 초기화 버튼을 누르면 서버에서도 삭제.

**중복 구매내역 감지** — 같은 날짜·물건명·금액이 여러 개 들어온 경우를 잡아낸다. 구매목록에서 중복 항목에 빨간 뱃지를 표시하고, "중복 N건" 버튼으로 한 번에 삭제할 수 있다. DB 레벨에도 unique constraint를 걸어서 아예 중복이 들어오지 못하게 막았다.

**내보내기** — 구매내역 전체를 CSV나 JSON으로 내보내는 기능을 추가했다. CSV는 Excel에서 한글이 깨지는 문제를 막기 위해 BOM(Byte Order Mark)을 앞에 붙였다.

그 외에 구매목록에 쇼핑몰별 필터 칩과 더 불러오기 페이지네이션도 추가했다.
