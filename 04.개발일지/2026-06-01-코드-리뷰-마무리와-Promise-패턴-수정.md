# 배포 첫날 — 에러 수정부터 에피소드/활동 구분까지

배포 첫날인데 쉬지 않았다. 로그인 500 에러를 고치고, 새 기능을 붙이고, 버그도 잡았다.

---

## 오늘 한 것

### 이메일 로그인 & Supabase 저장

이메일+비밀번호 로그인, localStorage → Supabase 에피소드 저장 전환, 히스토리 수정 + ICS 재다운로드를 오늘 처음 배포했다. Next.js 16에서 `middleware.ts`를 `proxy.ts`로 바꾸는 작업도 있었는데, 파일 이름만 바꾸면 되는 줄 알았다가 함수 이름까지 같이 바꿔야 한다는 걸 빌드 에러로 알았다.

### Vercel 500 에러 수정

배포 직후 서버에서 Internal Server Error가 났다. 로컬에서는 잘 됐는데 서버에서만 터지는 전형적인 환경변수 문제였다. 브라우저 콘솔에서 이 에러가 떴다:

```
@supabase/ssr: Your project's URL and API key are required to create a Supabase client!
```

`NEXT_PUBLIC_SUPABASE_URL`과 `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 Vercel에 없었다. 두 값을 Vercel 환경변수로 추가하고 재배포했다.

동시에 `proxy.ts`에 try/catch가 없어서 Supabase 연결 오류가 나면 500이 그대로 나가는 구조도 수정했다. 이제 Supabase 오류 시에도 `/login`으로 리다이렉트한다.

### 에피소드 / 활동 구분 기록

사용자가 "에피소드 등록은 종일 이벤트에만 해당되는데, 에피소드가 아닌 경우엔 시간을 지정해서 활동 내용을 쓰고 싶다"고 했다.

폼 상단에 **에피소드 등록** 토글을 추가했다.

- **ON**: 에피소드 번호·지역·장소·여행 입력 → 종일 ICS → Supabase 저장
- **OFF**: 시간(필수)·활동 내용 입력 → 시간 지정 ICS → Supabase 저장 안 함

### 버그 수정 — `isEpisode` 판별 오류

마무리 루틴 코드 리뷰에서 버그를 발견했다. `if (!episode.isEpisode)`로 체크하면 기존 DB 에피소드(`isEpisode: undefined`)도 비에피소드 경로로 들어간다. JavaScript에서 `undefined`는 falsy라서 생기는 문제다.

`if (episode.isEpisode === false)`로 바꿔서 `undefined`와 `false`를 구분하도록 수정했다.

## 막혔던 부분

Vercel 500 에러가 가장 당황스러웠다. 로컬에서 잘 되니 코드 문제가 아닌 건 알았는데, Supabase 클라이언트 자체가 환경변수 없이 실패하면서 proxy가 아무런 에러 처리 없이 500을 내뱉고 있었다. 브라우저 콘솔 에러를 보고 나서야 원인을 바로 잡았다.

`!episode.isEpisode` vs `episode.isEpisode === false`는 작은 차이지만 히스토리에 있는 134개 에피소드 전체의 ICS 다운로드가 잘못된 형식으로 생성될 뻔했다.
