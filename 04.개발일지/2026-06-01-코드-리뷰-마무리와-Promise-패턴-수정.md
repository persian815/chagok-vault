# 코드 리뷰 마무리와 Promise 패턴 수정

마무리 루틴을 돌렸더니 숨어있던 버그가 하나 나왔다.

---

## 오늘 한 것

오늘은 새 기능 개발보다는 그동안 쌓인 코드를 제대로 점검하는 날이었다. `/skills-end` 마무리 루틴을 돌렸고, 코드 리뷰 → 빌드 체크 → 문서 동기화 → 커밋 & 푸시 순서로 진행했다.

코드 리뷰에서 한 가지 패턴 문제를 발견했다. `app/api/chat/route.ts` 파일에서 채팅 기록을 DB에 저장할 때 "fire-and-forget" 방식을 쓰는 부분이 있는데, 원래 코드가 이렇게 생겼었다:

```typescript
supabase.from('chat_messages').insert([...]).then(() => {});
```

응답을 기다리지 않고 그냥 흘려보내는 게 목적인데, `.then(() => {})` 만으로는 실패했을 때 에러가 처리되지 않는 문제가 있다. Node.js에서 잡히지 않는 Promise 거절(unhandled rejection)이 발생하는 것이다.

처음엔 `.catch(() => {})`로 단순하게 바꾸려 했는데 Supabase의 쿼리 빌더 타입이 `.catch()`를 지원하지 않아 TypeScript 에러가 났다. 그래서 `Promise.resolve()`로 한 번 감싼 다음 `.catch()`를 붙이는 방식으로 해결했다:

```typescript
Promise.resolve(
  supabase.from('chat_messages').insert([...])
).catch(() => {});
```

TypeScript 타입 체크도 통과하고, 빌드도 깔끔하게 성공했다.

## 막혔던 부분

처음에 `.catch()`로 바꿨다가 `Property 'catch' does not exist on type 'PostgrestFilterBuilder'` 에러가 났다. Supabase 쿼리 빌더가 표준 Promise가 아니라 자체 타입이라 `.then()`은 있어도 `.catch()`가 없는 구조였다. `Promise.resolve()`로 네이티브 Promise로 변환하면 자연스럽게 해결된다는 걸 알았다.
