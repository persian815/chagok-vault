## 오늘 한 것

### Vercel 배포 및 버그 수정

로컬에서만 돌리던 알뜰살뜰 앱을 Vercel에 올렸다. 배포 자체는 GitHub 레포 연결하고 환경변수 넣고 끝. 문제는 그 다음이었다.

**이미지 업로드 500 에러.** Vercel의 기본 요청 크기 제한 4.5MB를 초과한 것. `next.config.mjs`에 10MB로 늘려서 해결했다.

**Cerebras 모델 이름 삽질.** `llama-3.3-70b` → 404. `llama3.3-70b` → 404. API에 직접 찔러보고 나서야 `llama3.1-8b`가 안정적으로 동작한다는 걸 확인했다. 공식 문서보다 직접 찔러보는 게 빨랐다.

**토큰 한도 초과.** AI 질문 시 구매내역 전체를 다 넣다 보니 `context_length_exceeded` 에러가 났다. 최근 3개월 + 150건으로 제한해서 해결.

**Vercel 자동 배포 12회.** 코드 수정마다 push했더니 배포가 쌓였다. 앞으로는 작업 중엔 커밋만, 마무리할 때 한 번만 push하는 규칙을 만들었다.

그 외 모바일 핀치줌 비활성화, 미들웨어 환경변수 null 체크, 설정 화면 배포 일시 표시도 챙겼다.

### 게스트 로그인 기능 추가

"게스트로 둘러보기" 버튼을 추가했다. 누르면 Supabase 익명 인증(Anonymous Auth)으로 임시 계정이 자동 생성되면서 앱 안으로 들어간다.

게스트 권한 정책:
- AI 질문, 이미지 분석 ✅
- 지식 추가/삭제, 프롬프트 편집, Storage 동기화 ❌
- 구매내역은 RLS로 익명 계정 ID 기준 분리되어 오너 데이터와 섞이지 않음

처음엔 `anonymous_provider_disabled` 에러가 뜨는데 메시지를 숨겨놔서 원인을 못 찾았다. 실제 에러를 보여줬더라면 Supabase 대시보드에서 Anonymous 토글 하나 켜는 것으로 5초 만에 해결됐을 텐데. 에러는 원문을 보여줘야 한다.

### 옵시디언 + GitHub 자동 동기화 설정

설정 화면에 블로그 탭을 만들었다. `persian815/blog` 레포의 `posts/` 폴더에서 .md 파일을 읽어오는 방식이다.

Obsidian Git 플러그인을 설치하면 10분마다 자동으로 commit + push된다. 옵시디언에서 글을 쓰면 앱 블로그 탭에서 바로 읽을 수 있는 구조가 완성됐다.

### 브랜드 이름 확정

기술적인 것 외에 브랜드 방향도 이날 최종 정리했다.
