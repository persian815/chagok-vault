# CHAGOK 작업공간 대정리

오늘은 코드 한 줄 안 짰다. 대신 작업 공간 전체를 뜯어고쳤다.

---

## 오늘 한 것

`blog`라는 이름으로 쓰던 옵시디언 볼트를 `vault`로 바꾸고, 코드 프로젝트들을 담을 `projects/` 폴더를 새로 만들었다. 기존에 `chagok project` 안에 있던 알뜰살뜰(littleRecords)도 `projects/littleRecords/`로 정리했다.

볼트 내부도 손봤다. 폴더가 뒤죽박죽이었는데, 번호를 붙여서 순서를 잡았다.

```
vault/
├── 00_브랜드/
├── 01_공통지식/
├── 02_프로젝트/
│   └── 알뜰살뜰/
├── 03_템플릿/
├── 04_발행글/
├── 05_요청/
└── 06_응답/
```

`posts`, `request`, `response`처럼 영어로 쓰던 폴더 이름도 한글로 바꿨다. 발행글, 요청, 응답. 훨씬 자연스럽다.

알뜰살뜰 프로젝트 폴더에는 개요, 설계만 있었는데 기능목록, 트러블슈팅, 회고 파일도 추가했다. 나중에 두 번째 프로젝트를 시작할 때 같은 구조로 시작할 수 있도록 `03_템플릿/새프로젝트-폴더-템플릿.md`도 만들어뒀다.

클로드 커맨드 스킬도 정리했다. 볼트 관련 스킬(발행글 푸시, 요청 처리)은 `~/.claude/commands/`로 옮겨서 어디서나 쓸 수 있게 하고, 개발일지와 클로드 일기처럼 프로젝트 맥락이 필요한 스킬은 `littleRecords/.claude/commands/`에 그대로 뒀다.

## 막혔던 부분

발행글 폴더를 `02_프로젝트/알뜰살뜰/` 아래에 둘지, 최상위에 둘지 잠깐 고민했다. 글의 성격을 보니 알뜰살뜰 전용 글이 아니라 클로드 일기처럼 프로젝트 무관한 글도 섞여 있어서, 결국 독립 폴더(`04_발행글/`)로 분리했다. 블로그가 프로젝트를 가리지 않는 게 맞는 것 같다.

## 다음에 할 것

구조는 잡혔으니 이제 내용을 채울 차례다. `00_개요.md`와 `01_설계.md`에 실제 내용을 정리하고, 트러블슈팅이랑 회고도 조금씩 쌓아가면 나중에 두 번째 프로젝트 시작할 때 훨씬 수월할 것 같다.
