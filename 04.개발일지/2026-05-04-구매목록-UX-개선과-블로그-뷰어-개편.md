# 구매목록 UX 개선과 블로그 뷰어 개편

쇼핑몰 필터, 중복 제거, 블로그 카테고리까지 — 오늘 하루도 정신없이 달렸다.

---

## 오늘 한 것

### 구매목록 기능 개선

구매목록에 기존 카테고리 필터(가전, 가구…)만 있었는데, 오늘 **쇼핑몰 필터**를 추가했다. 쿠팡이면 쿠팡, 네이버면 네이버 — 탭처럼 누르면 해당 쇼핑몰 구매내역만 보인다. 데이터에 있는 쇼핑몰만 자동으로 버튼이 생기는 방식이라 별도 설정 없이도 깔끔하게 동작한다.

**중복 항목 표시**도 넣었다. 같은 날짜에 같은 상품을 같은 금액으로 두 번 올리면 목록에 복사 아이콘이 뜬다. 여기서 멈추지 않고, 추가 버튼 옆에 **중복 N건 제거** 버튼도 만들었다. 누르면 중복 중 첫 번째만 남기고 나머지를 한꺼번에 삭제한다. 삭제 중엔 스피너가 돌아서 "지금 처리 중이에요"라는 피드백도 된다.

### 블로그 뷰어 + vault 구조 개편

설정 탭의 블로그 뷰어가 GitHub API 404 오류를 내고 있었다. 레포 이름이 `persian815/blog`에서 `persian815/chagok-vault`로 바뀐 게 원인이었다. 경로 수정 후 정상화.

이 김에 블로그 카테고리 구조도 전면 개편했다. 기존 `04_발행글` 폴더를 `04.개발일지`와 `05.클로드일기`로 분리하고, 요청/응답도 `06.아이디어요청` / `07.아이디어답변`으로 이름을 정리했다. vault 폴더도 실제로 이동 처리했다.

앱에서는 블로그 탭을 **전체 / 블로그 / 아이디어** 세 탭으로 재편했다. 블로그 탭에는 개발일지와 클로드일기가 날짜 기준으로 1:1 매핑되어 함께 보이고, 아이디어 탭은 요청 글을 누르면 응답 글이 바로 아래 이어지는 구조다.

### 마무리 루틴 개선

`/skills-end` 스킬에 5단계 `/skills-blog` 호출을 추가했다. 이제 코드 리뷰 → 빌드 → 문서 → 커밋 → 블로그 발행이 한 번의 명령으로 이어진다.

## 막혔던 부분

Next.js route 파일에서 배열 상수를 `export`하니 타입 에러가 났다. 라우트 파일은 HTTP 핸들러와 타입만 내보낼 수 있다는 Next.js 제약인데, 처음엔 원인을 바로 못 찾아서 잠깐 헤맸다.

## 다음에 할 것

블로그 뷰어 아이디어 탭에서 요청·응답 매칭이 `_답변` 파일명 패턴 기반인데, 나중에 파일명 규칙이 바뀌면 매칭이 깨질 수 있다. 지금은 규칙이 일관되니 괜찮지만 언젠가 더 견고하게 개선할 여지가 있다.
