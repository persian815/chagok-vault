## 프로젝트 구조

```
app/
├── (auth)/login/         ← 로그인
├── (app)/                ← 인증 필요 영역
│   ├── dashboard/        ← 메인 대시보드
│   ├── upload/           ← 이미지 업로드·파싱
│   ├── purchases/        ← 구매내역 목록
│   ├── ai-chat/          ← AI 질문
│   └── settings/         ← 설정 (프롬프트·블로그·로그 등)
└── api/                  ← Route Handlers
    ├── analyze/          ← Gemini 이미지 분석
    ├── chat/             ← Cerebras 텍스트 Q&A
    ├── purchases/        ← 구매내역 CRUD
    ├── knowledge/        ← 지식문서 관리
    ├── blog/             ← GitHub 블로그 포스트
    ├── logs/             ← AI·에러 로그
    └── prompts/          ← 프롬프트 CRUD
```

---

## DB 스키마

| 테이블 | 용도 |
|---|---|
| `purchases` | 구매내역 (date, shop, item, price, discount 등) |
| `knowledge_docs` | 지식문서 (title, category, content, input_type) |
| `ai_logs` | AI 호출 로그 (type, model, status, input, output) |
| `error_logs` | 에러 로그 |
| `prompts` | 사용자 커스텀 프롬프트 (key, content) |

모든 테이블: RLS 활성화, `auth.uid() = user_id` 정책.

## 핵심 설계 결정 (ADR)

### ADR-001: AI 이원화
- **결정:** 이미지 분석은 Gemini, 텍스트 Q&A는 Cerebras
- **이유:** Gemini는 이미지 분석 성능 좋음. Cerebras는 텍스트 대화 완전 무료
- **날짜:** 2026-05-01

### ADR-002: 프롬프트 시스템
- **결정:** DB 우선, 파일 fallback 구조
- **이유:** 사용자가 프롬프트 직접 편집 가능하면서 기본값 유지
- **구현:** `getPrompt(supabase, key, userId)` → DB 없으면 `lib/prompts/*.md`
- **날짜:** 2026-05-01

### ADR-003: 로깅 설계
- **결정:** logAI/logError에 supabase 인스턴스를 직접 넘기는 방식
- **이유:** 내부에서 createClient() 재생성 시 응답 후 컨텍스트 소멸로 RLS 실패
- **날짜:** 2026-05-01

### ADR-004: 게스트 로그인
- **결정:** Supabase Anonymous Auth 사용
- **이유:** 별도 계정 없이 앱 체험 가능. 게스트 데이터는 익명 ID로 자동 분리
- **게스트 허용:** AI 질문, 이미지 분석
- **게스트 차단:** 지식 추가/삭제, 프롬프트 편집, 수동 구매 추가, Storage 동기화
- **날짜:** 2026-05-02
