## DB 스키마

| 테이블 | 용도 |
|---|---|
| `couples` | 커플 정보 (wedding_date, name1, name2) |
| `tasks` | 할 일 (month, priority, assignee, title, detail, completed) |
| `events` | 일정 (date, title, assignee, type, memo) |

### couples
```sql
id            uuid PRIMARY KEY
user1_id      uuid REFERENCES auth.users
user2_id      uuid REFERENCES auth.users (nullable — 초대 전)
wedding_date  date NOT NULL
name1         text
name2         text
created_at    timestamptz DEFAULT now()
```

### tasks
```sql
id            uuid PRIMARY KEY DEFAULT gen_random_uuid()
couple_id     uuid REFERENCES couples(id)
title         text NOT NULL
month         text CHECK (month IN ('6월','7월','8월','9월'))
priority      text CHECK (priority IN ('최우선','중요','진행'))
assignee      text CHECK (assignee IN ('성룡','주영','같이'))
detail        text
completed     boolean DEFAULT false
completed_at  timestamptz
created_at    timestamptz DEFAULT now()
```

### events
```sql
id            uuid PRIMARY KEY DEFAULT gen_random_uuid()
couple_id     uuid REFERENCES couples(id)
title         text NOT NULL
date          date NOT NULL
assignee      text CHECK (assignee IN ('성룡','주영','같이'))
type          text  -- '예식장','촬영','피팅','미팅','예약','발송','기타'
memo          text
created_at    timestamptz DEFAULT now()
```

모든 테이블: RLS 활성화, `couple_id` 기반 접근 제어.

## 핵심 설계 결정 (ADR)

### ADR-001: 커플 공유 모델
- **결정:** `couples` 테이블에 user1·user2 두 UUID 저장, 모든 데이터는 couple_id로 묶음
- **이유:** 한 사람이 만들고 상대방을 초대링크로 연결하는 방식. 데이터는 couple 단위로 격리
- **초대 흐름:** user1 가입 → couple 생성 → 초대 링크 공유 → user2 가입 시 couple에 연결
- **날짜:** 2026-06-14

### ADR-002: Supabase Realtime으로 실시간 동기화
- **결정:** tasks·events 테이블에 Supabase Realtime 구독 설정
- **이유:** 성룡이 체크하면 주영 화면에 별도 새로고침 없이 바로 반영
- **구현:** `useEffect`에서 `supabase.channel('tasks').on('postgres_changes', ...)` 구독
- **날짜:** 2026-06-14

### ADR-003: 초기 데이터 시드
- **결정:** couple 생성 시 tasks 48개 자동 seed insert
- **이유:** 매번 할 일을 직접 입력하는 게 아니라 미리 준비된 체크리스트로 시작
- **구현:** `lib/seed/tasks.ts` 에 초기 데이터 배열 → couple 생성 API에서 일괄 insert
- **날짜:** 2026-06-14

### ADR-004: 모바일 우선 UI
- **결정:** 하단 탭 네비게이션 (홈·캘린더·타임라인·체크리스트)
- **이유:** 이동 중에도 두 사람이 스마트폰으로 자주 확인하는 앱
- **구현:** `app/layout.tsx` 하단에 BottomNav 컴포넌트 고정
- **날짜:** 2026-06-14
