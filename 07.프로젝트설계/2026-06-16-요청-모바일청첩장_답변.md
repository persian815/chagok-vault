## 답변

### 기술 스택 추천

기존 CHAGOK 스택을 그대로 활용할 수 있다.

| 분류 | 선택 | 이유 |
|---|---|---|
| Framework | Next.js (App Router) | 이미 익숙, SEO 필요 없으니 Client Component 중심 가능 |
| Styling | Tailwind CSS + 커스텀 CSS | 섹션별 애니메이션은 CSS Keyframe이 편함 |
| DB | Supabase (방명록) | 이미 두리 프로젝트와 동일 계정 활용 가능 |
| Storage | Supabase Storage (게스트스냅) | 두리와 동일 |
| 배포 | Vercel | GitHub push 자동 배포 |
| 참석 여부 | 구글 폼 외부 링크 | 별도 구현 불필요, 골드 버튼으로 외부 연결 |

### 섹션별 구현 핵심

#### S01 히어로 커버
```tsx
// 전체화면 배경 + 세리프 타이포
<section className="h-screen relative">
  <img className="object-cover w-full h-full" />
  <div className="absolute inset-0 flex flex-col items-center justify-center">
    <p className="font-playfair text-white">our wedding day</p>
    <p>2026.09.19</p>
  </div>
</section>
```
배경음악은 `<audio>` 태그 + useRef로 제어. 자동재생은 브라우저 정책상 막히므로 **팝업 닫기 버튼 클릭 시 재생 시작**하면 자연스럽다.

#### S04 웨딩 인터뷰 (바텀시트)
```tsx
// 바텀시트 = fixed + translate-y 애니메이션
const [open, setOpen] = useState(false)
<div className={`fixed bottom-0 inset-x-0 transition-transform ${open ? 'translate-y-0' : 'translate-y-full'}`}>
```

#### S05 달력 (혼례일 골드 원)
외부 라이브러리 없이 직접 구현 권장. 9월 달력을 정적으로 렌더링, 19일 셀에만 골드 원 스타일.

#### S07 지그재그 타임라인
```
홀수 항목: 사진(왼) ↔ 텍스트(오)
짝수 항목: 텍스트(왼) ↔ 사진(오)
중앙: 세로선 + 골드 점
```
Intersection Observer로 스크롤 진입 시 fade-in 애니메이션 적용.

#### S08 갤러리 라이트박스
3열 그리드 클릭 시 전체화면 오버레이. 스와이프는 `touchstart`/`touchend` 이벤트로 직접 구현 또는 `swiper` 라이브러리 사용.

#### S13 계좌 복사
```tsx
navigator.clipboard.writeText(accountNumber)
  .then(() => toast('복사되었습니다'))
```
카카오페이 버튼은 카카오페이 송금 링크(`https://qr.kakaopay.com/...`) 외부 링크로 연결.

#### S15 함께한 시간 (초 단위 카운터)
```tsx
const [elapsed, setElapsed] = useState(0)
useEffect(() => {
  const start = new Date('2022-12-24').getTime()
  const timer = setInterval(() => {
    setElapsed(Math.floor((Date.now() - start) / 1000))
  }, 1000)
  return () => clearInterval(timer)
}, [])
```

### 백엔드 구현

#### 방명록 DB 스키마
```sql
CREATE TABLE wedding_guestbook (
  id         uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name       text NOT NULL,
  message    text NOT NULL,
  created_at timestamptz DEFAULT now()
);
-- RLS 없이 공개 (누구나 읽기·쓰기 가능)
-- 삭제는 관리자만: service_role key 사용
```

API 라우트:
- `GET /api/guestbook` — 전체 조회 (최신순)
- `POST /api/guestbook` — 이름 + 메시지 저장
- `DELETE /api/guestbook/[id]` — service_role key로만 삭제 가능

#### 게스트스냅 Storage
```
버킷명: guest-snap
정책: 인증 없이 업로드 허용 (혼례일 전까지 비활성은 앱에서 날짜 체크)
파일명: {timestamp}_{originalName}
```
혼례일(2026-09-19) 이전 비활성:
```tsx
const isActive = new Date() >= new Date('2026-09-19')
if (!isActive) return <p>혼례일 당일부터 업로드 가능합니다</p>
```

### 진입 팝업 (오늘하루 안보기)

```tsx
useEffect(() => {
  const hidden = localStorage.getItem('popup-hidden-date')
  if (hidden === today) setVisible(false)
}, [])

const handleHide = () => {
  localStorage.setItem('popup-hidden-date', today)
  setVisible(false)
  audioRef.current?.play() // 팝업 닫을 때 음악 시작
}
```

### 네이버 지도 임베드

네이버 지도는 iframe 임베드 방식 사용:
```html
<iframe
  src="https://map.naver.com/p/entry/place/{장소ID}"
  width="100%" height="300"
/>
```
장소 ID는 네이버 지도에서 해당 웨딩홀 검색 후 URL에서 확인.
