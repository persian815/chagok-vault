# 배포하고, 삽질하고, 블로그까지

알뜰살뜰 앱을 처음으로 Vercel에 배포한 날. 예상보다 훨씬 많은 일이 있었다.

---

## 오늘 한 것

오늘 목표는 단순했다. 로컬에서만 돌리던 앱을 Vercel에 올려서 아이폰에서도 쓸 수 있게 만드는 것.

배포 자체는 금방 됐다. GitHub 레포 연결하고 환경변수 넣고 Deploy 버튼 누르면 끝이니까. 문제는 그 다음이었다.

**이미지 업로드가 500 에러를 냈다.** 알고 보니 Vercel의 기본 요청 크기 제한이 4.5MB인데 스마트폰 캡처 이미지가 그걸 넘었다. `next.config.mjs`에 10MB로 늘려서 해결.

**AI 질문이 400 에러를 냈다.** Gemini는 됐는데 Cerebras가 안 됐다. 에러 메시지를 제대로 안 보여주고 있었던 게 문제였다. 로그를 열어보니 `context_length_exceeded` — 구매내역을 전부 AI에 넣다 보니 토큰이 넘쳤다. 최근 3개월치만 넣도록 제한해서 해결.

**모델도 삽질했다.** `llama-3.3-70b`로 바꿨더니 404. 모델 이름이 `llama3.3-70b`인 줄 알았는데 그것도 없었다. Cerebras API에서 직접 확인하니 결국 `llama3.1-8b`가 가장 안정적이었다.

**Vercel 배포가 12번 쌓였다.** 코드 수정할 때마다 GitHub에 push했더니 자동 배포가 계속 돌았다. 앞으로는 작업 중엔 커밋만 하고 마무리할 때 한 번만 push하기로 규칙을 만들었다.

그 외에 모바일 핀치줌 비활성화, 미들웨어 환경변수 null 체크, 설정 화면에 배포 일시 표시 등 자잘한 것들도 챙겼다.

## 막혔던 부분

Cerebras 모델 이름 찾는 게 제일 오래 걸렸다. 공식 문서보다 직접 API를 찔러보는 게 더 빨랐다.

## 오늘 확정한 것 하나 더

기술적인 것과 별개로, 오늘 브랜드 방향도 최종 정리했다.

**브랜드명: 차곡(CHAGOK)**. 차곡차곡 쌓인다는 한국적인 감성. MUJI처럼 의미보다 발음 중심으로 쓰기로 했다. CHAGO는 자동차 이미지가 연상된다고 해서 제외하고, CHAGOK으로 확정.

**프로젝트 영문명: Little Records**. Little Things, Daily Keep, Gather 등을 비교했는데, "생활 기록을 쌓는다"는 철학에 가장 잘 맞는 이름이었다. 알뜰살뜰 = Little Records.

브랜드 구조는 이렇게 됐다:
- 브랜드: **차곡 (CHAGOK)**
- 첫 번째 프로젝트: **알뜰살뜰 / Little Records**

나중에 일정 공유, 부부 루틴 기록, 생활 데이터 관리 같은 프로젝트들이 차곡 브랜드 아래 붙을 예정이다.
