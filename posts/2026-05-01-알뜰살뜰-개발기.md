# 알뜰살뜰 개발기 — AI 가계부 앱을 바이브 코딩으로 만들다

42살에 처음으로 바이브 코딩 방식으로 풀스택 웹앱을 만들어봤다. 아내와 함께 쓸 가계부인데, 쇼핑몰 주문 캡처 이미지를 찍으면 AI가 알아서 파싱해주는 앱이다.

---

## 왜 만들었나

신혼집에 들어오면서 구매내역이 순식간에 쌓였다. 무인양품, 쿠팡, 네이버쇼핑, 다이소, 오늘의집, SSG, 이구홈 등 9개 쇼핑몰에서 162건, 실결제 약 1,061만 원. 처음엔 엑셀로 정리했다. 스크린샷 찍고, 수동으로 입력하고, 쇼핑몰별로 시트 나누고. v5까지 만들고 나서야 "이건 계속 못 하겠다"는 결론이 났다.

"캡처 이미지 하나만 올리면 AI가 알아서 파싱해주는 앱이 있으면 어떨까" — 그게 알뜰살뜰의 시작이었다.

시중 가계부 앱들은 기능이 너무 많거나, 우리 패턴과 맞지 않거나, 광고가 많았다. 그래서 직접 만들기로 했다.

---

## 기술 스택 선택

처음부터 복잡하게 가지 않기로 했다. 부부 2인만 쓰는 개인 앱이니까.

| 분류 | 선택 | 이유 |
|---|---|---|
| Frontend | Next.js 14 + TypeScript | App Router로 서버/클라이언트 깔끔하게 분리 |
| Styling | Tailwind + shadcn/ui | 빠르게 모바일 UI 만들기 |
| DB/Auth | Supabase | RLS로 인증 붙이기 쉬움, 무료 플랜 충분 |
| 이미지 AI | Google Gemini 2.5 Flash | 이미지 분석 성능 좋고 비용 저렴 |
| 텍스트 AI | Cerebras | 완전 무료, 빠름 |
| 배포 | Vercel | GitHub push만 하면 자동 배포 |

---

## 주요 기능

### 1. 이미지 → 자동 파싱

쇼핑몰 주문 완료 화면을 캡처해서 올리면 Gemini가 물건명, 금액, 날짜, 쇼핑몰을 읽어낸다. 결과를 화면에서 확인하고 수정한 뒤 저장하는 방식이다.

처음엔 Gemini 응답이 JSON으로 깔끔하게 오지 않고 앞뒤에 설명 텍스트가 붙어오는 경우가 있었다. `[` ~ `]` 구간만 추출하는 파싱 로직으로 해결했다.

### 2. AI 질문 (가계부 Q&A)

"이번 달 얼마 썼어?", "가장 많이 산 카테고리는?" 같은 자연어 질문을 하면 구매내역을 기반으로 답해준다. Cerebras의 Llama 3.1 8B 모델을 쓰는데 완전 무료다.

초반에 토큰 한도 초과 문제가 있었다. 전체 구매내역을 다 넣다 보니 8192 토큰을 넘어버렸다. 최근 3개월 + 150건으로 제한해서 해결했다.

### 3. 지식베이스

브랜드 정보, 예산 계획 같은 문서를 `.md` 파일로 올리면 Gemini가 요약해서 AI 답변에 반영한다. "무인양품 제품 특징 알려줘" 같은 질문도 가능해진다.

### 4. 대시보드

카테고리별 도넛 차트, 월별 트렌드 막대 차트로 소비 패턴을 한눈에 볼 수 있다.

## 삽질 모음

### Gemini JSON 파싱 실패

`maxOutputTokens: 2048`이 너무 작아서 응답이 중간에 잘렸다. 8192로 올리고, JSON 배열 구간만 추출하는 로직을 추가했다.

### Cerebras 모델 삽질

처음에 `qwen-3-235b-a22b-instruct-2507`을 기본값으로 설정했다가 400 에러. `llama-3.3-70b`로 바꿨더니 404. Cerebras API에서 지원하는 모델 목록을 직접 확인하고 나서야 `llama3.1-8b`가 안정적으로 동작하는 걸 알았다.

### Vercel 자동 배포 남용

코드를 수정할 때마다 GitHub에 push했더니 45분 동안 배포가 12번 쌓였다. 이후로는 작업 중엔 commit만 하고, 마무리할 때 한 번만 push하는 규칙을 만들었다.

### MIDDLEWARE_INVOCATION_FAILED

Vercel 배포 후 미들웨어에서 500 에러가 났다. 환경변수가 없을 때 `createServerClient`가 크래시하는 문제였다. null 체크를 추가해서 해결했다.

## 바이브 코딩 방식

이번 프로젝트는 Claude Code와 함께 작업했다. 직접 코드를 짜는 게 아니라 "이런 기능 만들어줘", "이 오류 고쳐줘" 하면서 대화로 진행했다.

마무리 루틴도 슬래시 커맨드로 자동화했다:

```
/skills-cr    → 보안·패턴 검토 및 자동 수정
/skills-build → TypeScript·빌드 확인
/skills-docs  → CLAUDE.md 문서 동기화
/skills-push  → 커밋 & 푸시
```

코드를 직접 이해하지 못해도 앱이 돌아가는 게 신기하면서도, 오류가 났을 때 원인을 파악하는 능력이 점점 늘고 있다는 느낌이 든다.

## 브랜드 방향

앱을 만들면서 동시에 이 프로젝트가 어떤 브랜드로 이어질지도 함께 고민했다. 단순한 가계부 앱이 아니라, 기록·정리·생활 데이터를 차곡차곡 쌓는 서비스로 키우고 싶었다. 오늘의집, 무인양품 같은 생활 브랜드 감성이 목표였다. 영어 기반 SaaS 스타일보다 한국적이고 담백한 방향.

브랜드 이름은 **차곡(CHAGOK)**. 차곡차곡 — 기록이 쌓이고, 데이터가 축적되고, 생활이 정돈되는 의미다. MUJI처럼 의미보다 발음 중심으로 CHAGOK을 쓰기로 했다.

알뜰살뜰은 차곡 브랜드의 첫 번째 프로젝트다. 영문명은 **Little Records** — 일상의 작은 기록을 쌓는다는 철학이 담긴 이름이다. 앞으로 일정 공유, 부부 생활 관리, 루틴 기록 같은 다른 프로젝트들도 차곡이라는 브랜드 아래 이어질 예정이다.
