# 시놀리지 포토 관리 앱 — 프로젝트 설계

> 요구사항 파일: 2026-06-25-요청-시놀리지포토.md
> 설계 날짜: 2026-06-25

## 프로젝트 개요

**SynoPhoto Manager** — 시놀리지 NAS에 저장된 사진을 웹에서 관리·정리하는 개인용 도구.  
중복/유사 사진 제거, EXIF 기반 파일명 변경, 연도·월 폴더 재정리 기능을 제공한다.  
타겟: 시놀리지 NAS를 운영하는 개인 사용자 (1인).

---

## 기술 스택

| 분류 | 선택 | 이유 |
|---|---|---|
| Frontend | Next.js 15 + TypeScript | 검증된 스택. App Router로 서버·클라이언트 분리 |
| Styling | Tailwind + shadcn/ui | 빠른 UI, 모바일 대응 |
| DB/Auth | **없음 (Supabase 불필요)** | 데이터 소스가 시놀리지 NAS 자체 — 별도 DB 불필요 |
| 시놀리지 연동 | Synology DSM REST API (SYNO.Foto.*) | 공식 API로 사진 목록·다운로드·삭제·이동 |
| 이미지 해싱 | `sharp` + `imghash` (pHash) | 중복/유사 감지용 지각적 해시 |
| EXIF 파싱 | `exifr` | 브라우저·서버 모두 지원, 촬영일 추출 |
| 배포 | **로컬 실행 (개발 서버)** or Vercel (주의 필요) | 시놀리지 자격증명이 포함되므로 로컬 우선 |

> 추론: Supabase가 필요 없는 첫 번째 차곡 프로젝트. 시놀리지 NAS가 데이터베이스 역할을 함.
> 추론: 외부 Vercel 배포 시 자격증명 노출 위험 → 로컬 `npm run dev` 또는 내부망 배포 권장.

## 시놀리지 DSM API 핵심 엔드포인트

```
# 인증
POST /webapi/auth.cgi
  api=SYNO.API.Auth&version=7&method=login
  &account=USER&passwd=PASS&session=SynoPhoto&format=cookie

# 사진 목록
GET /webapi/entry.cgi
  ?api=SYNO.Foto.Browse.Item&version=1&method=list
  &folder_id=FOLDER_ID&offset=0&limit=100

# 폴더 목록
GET /webapi/entry.cgi
  ?api=SYNO.Foto.Browse.Folder&version=1&method=list

# 사진 삭제
POST /webapi/entry.cgi
  api=SYNO.Foto.Browse.Item&version=1&method=delete
  &id=[1,2,3]

# 사진 이동
POST /webapi/entry.cgi
  api=SYNO.Foto.Browse.Item&version=1&method=copy_to
  &id=[1]&target_folder_id=FOLDER_ID

# 썸네일
GET /webapi/entry.cgi
  ?api=SYNO.Foto.Thumbnail&version=2&method=get
  &id=PHOTO_ID&cache_key=KEY&type=unit&size=sm
```

## 핵심 설계 결정 (ADR)

### ADR-001: 서버사이드 프록시 방식
- **결정:** 브라우저에서 시놀리지 API 직접 호출 대신, Next.js API Routes를 프록시로 사용
- **이유:** CORS 문제 회피 + 자격증명을 서버에만 보관 (세션 쿠키)
- **구현:** `lib/synology/client.ts`에서 fetch로 DSM API 호출, 세션 토큰은 iron-session 쿠키

### ADR-002: 중복 감지 — 파일 해시 우선, pHash 보조
- **결정:** 완전 중복은 파일 MD5 해시로, 유사한 사진은 pHash (hamming distance ≤ 10)
- **이유:** MD5는 빠르고 정확, pHash는 리사이즈·필터 적용된 유사 사진도 잡음
- **구현:** 썸네일을 서버에서 받아 `sharp`로 8×8 그레이스케일 → 비트 해시 비교

### ADR-003: 스캔은 청크 단위 스트리밍
- **결정:** 전체 사진을 한 번에 스캔하지 않고 페이지(100장)씩 처리 후 SSE로 진행률 전송
- **이유:** 사진이 수천 장이면 단일 요청 타임아웃 위험
- **구현:** `GET /api/scan/duplicates` → `ReadableStream`으로 진행 이벤트 push

### ADR-004: 폴더 구조 규칙
- **결정:** 목표 폴더 구조 = `/개인공간/YYYY/YYYY-MM`
- **이유:** 현재 혼재된 구조를 EXIF 촬영일 기준으로 재정리
- **구현:** organize 페이지에서 "폴더 미분류" 파일 목록 → 이동 버튼

### ADR-005: 로컬 배포 우선
- **결정:** Vercel 배포 않고 `npm run dev`로 로컬 실행
- **이유:** 시놀리지 자격증명이 .env.local에 있어 외부 노출 위험
- **구현:** .env.local에 `SYNOLOGY_URL`, `SYNOLOGY_USER`, `SYNOLOGY_PASS` 저장
