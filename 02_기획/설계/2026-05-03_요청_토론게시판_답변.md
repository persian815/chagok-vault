# AI 페르소나 토론 앱 설계

> 요청 날짜: 2026-05-03  
> 답변 날짜: 2026-05-03

---

## 요청 요약

매일 운영자 에이전트가 주제를 정해 게시판에 올리면, 10명의 유명인 페르소나 에이전트가 순차적으로 댓글을 달고, 50개가 쌓이면 운영자가 토론을 요약·마감하는 AI 토론 앱을 설계한다.

---

## 답변

### 전체 구조 개요

```
[운영자 에이전트]
  └─ 매일 주제 생성 → 게시판 등록
        │
        ▼
[10명 페르소나 에이전트] (철학자, 예술가, 정치인 등)
  └─ 1번 에이전트 → 첫 글 작성
  └─ 2~10번 에이전트 → 앞 내용 읽고 댓글
  └─ 다시 1번부터 순환 … (총 50개)
        │
        ▼
[운영자 에이전트]
  └─ 토론 요약 댓글 → 종료
```

---

### 에이전트 구성

#### 운영자 에이전트 (1명)
- **역할**: 주제 생성, 토론 진행 관리, 최종 요약
- **주제 생성 기준**: 철학·사회·문화·시사 등 다양한 영역에서 논쟁 가능한 주제
- **예시 주제**: "자유와 안전 중 무엇이 더 중요한가?", "AI는 예술을 창조할 수 있는가?"

#### 페르소나 에이전트 (10명)
각 에이전트는 고유한 세계관, 말투, 논거 스타일을 가진다.

| # | 페르소나 | 성향 | 특징 |
|---|---------|------|------|
| 1 | 소크라테스 | 철학자 | 질문으로 논점을 파고드는 산파술 |
| 2 | 니체 | 철학자 | 강자의 의지, 도덕 비판 |
| 3 | 한나 아렌트 | 정치철학자 | 공공성·전체주의 경계 |
| 4 | 레오나르도 다빈치 | 예술가·과학자 | 감각적 관찰, 실험 중심 |
| 5 | 버지니아 울프 | 소설가 | 내면의 흐름, 개인의 서사 |
| 6 | 마키아벨리 | 정치인·사상가 | 현실주의, 목적이 수단을 정당화 |
| 7 | 마르크스 | 사상가 | 구조·계급 비판 |
| 8 | 마리 퀴리 | 과학자 | 데이터와 실증, 편견에 맞선 여성 |
| 9 | 간디 | 정치인·운동가 | 비폭력·도덕적 저항 |
| 10 | 오스카 와일드 | 작가 | 위트와 반전, 미학주의 |

---

### 토론 진행 흐름

```
1. 운영자: 주제 등록 (매일 자동)
2. 에이전트 #1: 주제에 대한 첫 의견 작성 (200~300자)
3. 에이전트 #2~10: 이전 댓글을 컨텍스트로 읽고 반응 (100~200자)
4. 다시 에이전트 #1부터 순환
5. 50번째 댓글 완료 시 운영자가 종합 요약 작성 → 토론 종료
```

- 각 에이전트는 **이전 전체 댓글**을 컨텍스트로 받아 흐름을 이어간다
- 에이전트마다 시스템 프롬프트에 페르소나를 정의해 일관성 유지

---

### 기술 설계

#### 데이터 구조 (DB)

```sql
topics       — id, content, date, status(active/closed), summary
comments     — id, topic_id, persona_id, content, order_num, created_at
personas     — id, name, description, system_prompt
```

#### API 흐름

```
POST /api/topics/generate     ← 운영자 에이전트 (매일 1회, cron)
POST /api/comments/next       ← 다음 순서 에이전트 댓글 생성
POST /api/topics/:id/close    ← 50번째 댓글 후 운영자 요약·종료
GET  /api/topics/:id          ← 게시판 조회
```

#### AI 호출 방식

```typescript
// 각 댓글 생성 시
const messages = [
  { role: "system", content: persona.system_prompt },
  { role: "user", content: `주제: ${topic}\n\n지금까지 댓글:\n${prevComments}` }
];
```

#### 자동화

- **매일 주제 생성**: Vercel Cron / GitHub Actions
- **댓글 자동 순환**: 이전 댓글 저장 후 다음 에이전트 호출 트리거
- 속도 조절: 댓글 간 1~2분 딜레이로 자연스러운 흐름 연출

---

### 프론트엔드 구성

```
/board                  ← 주제 목록 (날짜별)
/board/:id              ← 토론 상세 (댓글 스레드 형식)
  └─ 페르소나 아이콘 + 이름 표시
  └─ 실시간 업데이트 (Supabase Realtime 활용 가능)
/admin                  ← 운영자 대시보드 (주제 관리, 강제 종료 등)
```

---

### 고려사항

| 항목 | 내용 |
|------|------|
| 토큰 비용 | 댓글이 쌓일수록 컨텍스트 길어짐 → 최근 N개만 전달하거나 요약 압축 |
| 페르소나 일관성 | 시스템 프롬프트를 충분히 상세하게 작성 (말투, 주요 논점, 금기어 등) |
| 무한루프 방지 | 댓글 카운터를 DB에서 관리하고 50개 초과 시 API 차단 |
| 속도 | 50개 댓글을 실시간 생성 vs 미리 배치 생성 후 노출하는 방식 선택 |

## 다음 단계

1. **페르소나 시스템 프롬프트 작성** — 각 10명의 특성을 구체적으로 정의
2. **DB 스키마 확정 및 마이그레이션** — topics, comments, personas 테이블
3. **운영자 에이전트 구현** — 주제 생성 + 요약 로직
4. **댓글 순환 엔진 구현** — 순서 관리 + 컨텍스트 전달
5. **게시판 UI 구현** — 페르소나 아이콘·이름 포함 스레드 뷰
6. **Cron 스케줄 설정** — 매일 자정 주제 자동 등록

구현 범위나 우선순위에 따라 MVP(최소 기능 버전)로 먼저 단일 주제 수동 실행부터 시작하는 것을 권장한다.
