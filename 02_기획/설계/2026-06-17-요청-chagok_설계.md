## 프로젝트 개요

**알뜰살뜰(소비) + dayOne(에피소드) + 두리(할일)를 하나의 앱으로 통합한 부부 일상 기록 허브.**

성룡 & 주영이 iPhone 홈 화면에서 바로 접근하고, 두 사람의 기록을 실시간으로 공유한다.
iCloud에 공개 API가 없으므로 chogok에 직접 입력 → 필요 시 ICS로 내보내는 방식을 사용한다.

## 기술 스택

| 분류 | 선택 | 이유 |
|---|---|---|
| Frontend | Next.js 15 (App Router) + TypeScript | 요구사항 명시, App Router 서버/클라이언트 분리 |
| Styling | Tailwind CSS 4 + shadcn/ui | 전 프로젝트 검증 완료, 모바일 UI 빠르게 |
| DB/Auth | Supabase PostgreSQL + Supabase Auth | 전 프로젝트 검증, RLS로 인증 간결 |
| Storage | Supabase Storage | 에피소드 사진·영수증 저장 |
| Realtime | Supabase Realtime | tasks/events 실시간 동기화 (두리에서 검증됨) |
| 이미지 AI | Gemini 2.5 Flash | 알뜰살뜰에서 검증, maxOutputTokens 8192 설정 필수 |
| 텍스트 AI | Cerebras llama3.1-8b | 완전 무료. 모델명 API 직접 확인 필수 |
| 배포 | Vercel | GitHub push 자동 배포 |
| PWA | next-pwa | manifest.json + Service Worker, 홈 화면 아이콘 |

## DB 스키마

### profiles
```sql
CREATE TABLE profiles (
  id         uuid PRIMARY KEY REFERENCES auth.users ON DELETE CASCADE,
  name       text,
  created_at timestamptz DEFAULT now()
);
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "본인만 접근" ON profiles FOR ALL USING (auth.uid() = id);
```

## 핵심 설계 결정 (ADR)

### ADR-001: 통합 앱 vs 기존 앱 유지

- **결정:** 세 앱을 chogok 하나로 통합. 기존 앱은 단계적으로 대체
- **이유:** 홈 캘린더가 세 도메인 데이터를 동시에 보여야 하므로 분리된 앱으로는 구현 불가능
- **구현:** 기존 Supabase 프로젝트의 데이터를 마이그레이션하거나 새 프로젝트 시작

### ADR-002: couples 테이블 제거 → user_id 기반 격리

- **결정:** 두리의 `couples` 테이블 구조를 버리고 `user_id` 기반 단순 격리
- **이유:** 부부 공유를 위해 같은 계정을 쓰거나 각자 계정으로 접속. 복잡한 커플 연결 로직 불필요
- **구현:** 모든 테이블 `user_id` 컬럼 + RLS. 공유 필요 시 동일 계정 사용

### ADR-003: 할일을 "프로젝트" 개념으로 확장

- **결정:** tasks가 `project_id`에 속하는 구조. 결혼 준비는 projects 중 하나
- **이유:** 결혼 후에도 신혼집 꾸미기, 여행 등 계속 사용 가능
- **구현:** 첫 로그인 시 "결혼 준비" 프로젝트 seed 자동 생성

### ADR-004: PWA (next-pwa)

- **결정:** React Native 대신 Next.js PWA
- **이유:** 기존 Next.js 스택 재활용, 웹 배포로 양쪽 다 접근 가능
- **구현:** `next-pwa` 패키지 + `manifest.json` + 아이콘 2종 (192, 512)
- **삽질 주의:** Tailwind 4는 설정 방식이 v3과 다름, next-pwa는 next 16 호환성 확인 필요

### ADR-005: 통합 캘린더 API

- **결정:** `/api/home/calendar` 단일 엔드포인트에서 3개 테이블 병렬 쿼리
- **이유:** 캘린더 렌더링 시 왕복 3번보다 1번이 성능상 유리
- **구현:**
  ```typescript
  const [episodes, events, purchases] = await Promise.all([
    supabase.from('episodes').select(...).gte('date', startDate).lte('date', endDate),
    supabase.from('events').select(...).gte('date', startDate).lte('date', endDate),
    supabase.from('purchases').select(...).gte('date', startDate).lte('date', endDate),
  ])
  ```

### ADR-006: 색상 팔레트 CSS 변수

- **결정:** 도메인별 색상을 CSS 변수로 관리
- **구현:**
  ```css
  :root {
    --brand:    #3D6B4F; /* 세이지 그린 — 브랜드, 오늘 날짜 */
    --episode:  #D86344; /* 코랄 — 에피소드 dot */
    --todo:     #B5825A; /* 로즈골드 — 할일 dot */
    --spending: #3D7D5A; /* 세이지 그린(진) — 소비 dot */
  }
  ```
