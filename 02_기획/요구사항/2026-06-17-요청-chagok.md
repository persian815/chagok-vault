## 1. 프로젝트 배경

"차곡차곡 우리의 기록을 쌓는다"는 철학으로, 부부(성룡 & 주영)가 일상을 기록하는 통합 허브 앱이다.

기존에 바이브코딩으로 각각 만들었던 세 앱을 하나로 통합한다:

| 기존 앱    | 역할                                             | 통합 방향                              |
| ---------- | ------------------------------------------------ | -------------------------------------- |
| 알뜰살뜰   | AI 소비 기록 (Gemini 이미지 파싱 + Cerebras Q&A) | 소비 탭으로 이전                       |
| dayOne     | 데이트 에피소드 기록 (EXIF 파싱, ICS 생성)       | 홈 캘린더 + 에피소드로 이전            |
| 두리 (WPC) | 결혼 준비 체크리스트·일정                        | 할일 탭 내 "결혼 준비" 프로젝트로 이전 |

현실 데이터는 iCloud(사진, 캘린더, 메모)에 있지만 iCloud에 공개 API가 없으므로, chogok에 직접 입력하고 필요 시 ICS로 iCloud에 내보내는 방식을 사용한다.

## 3. 핵심 기능 요구사항

### 3-1. 홈 — 통합 캘린더

- 에피소드, 할일 일정, 소비를 하나의 월별 캘린더에서 조망
- 날짜마다 3가지 컬러 dot으로 데이터 존재 여부 표시
  - 코랄 dot: 에피소드
  - 로즈골드 dot: 할일·일정
  - 세이지그린 dot: 소비
- 오늘 날짜: 브랜드 초록 원
- dot 있는 날짜 탭 → 바텀 시트(날짜 패널)로 에피소드/일정/소비 통합 표시
- 하단 배너: 진행 중인 할일 프로젝트 요약

### 3-2. 소비 탭

- 월별 총액 + 카테고리별 소계 카드
- 영수증·주문서 사진 업로드 → Gemini 2.5 Flash 자동 파싱 → 사용자 확인 후 저장
- 자연어 질문 ("이번 달 가구 지출 얼마야?") → Cerebras llama3.1-8b 답변
- 최근 기록 목록: 상점명, 품목명, 금액

### 3-3. 할일 탭 (핵심 설계 결정)

**탭 이름은 "할일"이다. 결혼 준비는 그 안의 하나의 프로젝트일 뿐이다.**

- 프로젝트 목록 화면: 카드로 각 프로젝트 표시 (아이콘, 이름, 기한, 진행률 바)
- 새 프로젝트 추가 가능 (결혼 후에도 신혼집 꾸미기, 여행 계획 등 계속 사용)
- 프로젝트 상세: 체크리스트 + 다가오는 일정
- 체크박스 탭 시 Supabase Realtime으로 상대방 화면에도 즉시 반영
- 결혼 준비 프로젝트: 첫 로그인 시 seed 데이터 자동 생성 (tasks 48개, events 7개)

### 3-4. FAB (+) 버튼

- 어느 화면에서든 우하단 원형 버튼
- 탭 시 바텀 시트로 3가지 옵션:
  1. 데이트 에피소드 (사진 EXIF → 날짜·장소 자동 입력)
  2. 소비 기록 (영수증 사진 → AI 파싱)
  3. 할일 추가 (프로젝트 선택 → 항목 추가)

### 3-5. 에피소드 (캘린더 연동)

- 사진 업로드 → EXIF에서 날짜·GPS 좌표 추출 → 주소로 변환
- 에피소드 저장 시 ICS 파일 자동 생성 → 사용자가 iCloud 캘린더에 수동 추가
- 에피소드 번호(Ep. N) 자동 부여

## 4. 네비게이션 구조

```
바텀 탭 4개: 홈 | 소비 | 할일 | +(FAB)

화면 계층:
홈 (/)
  └── 날짜 패널 (바텀 시트)
소비 (/spending)
  └── 소비 상세
할일 (/todos)
  └── 프로젝트 상세 (/todos/[id])
에피소드 (/episodes)   ← 탭에 없음, 캘린더에서 접근
로그인 (/login)
```

## 5. 기술 스택 (기존 프로젝트와 통일)

- Frontend: Next.js 15 (App Router) + TypeScript
- Styling: Tailwind CSS 4 + shadcn/ui
- DB/Auth: Supabase PostgreSQL + Supabase Auth
- Storage: Supabase Storage (에피소드 사진, 영수증)
- Realtime: Supabase Realtime (tasks, events 테이블)
- AI 이미지: Gemini 2.5 Flash
- AI 텍스트: Cerebras llama3.1-8b
- 배포: Vercel
- PWA: manifest.json + Service Worker

## 6. 데이터 모델 요약

기존 `couples` 테이블 구조 제거 → `projects` 테이블로 단순화

### 핵심 테이블

| 테이블           | 격리 기준  | 설명                                 |
| ---------------- | ---------- | ------------------------------------ |
| `profiles`       | user_id    | 사용자 프로필                        |
| `episodes`       | user_id    | 데이트 에피소드                      |
| `purchases`      | user_id    | 소비 기록                            |
| `knowledge_docs` | user_id    | AI 지식 문서                         |
| `prompts`        | user_id    | AI 프롬프트                          |
| `ai_logs`        | user_id    | AI 응답 로그                         |
| `projects`       | user_id    | 할일 프로젝트 (결혼 준비, 신혼집 등) |
| `tasks`          | project_id | 할일 항목                            |
| `events`         | project_id | 프로젝트 일정                        |

모든 테이블 RLS 활성화.

### 통합 캘린더 API

```typescript
// GET /api/home/calendar?year=YYYY&month=M
// 3개 테이블 병렬 쿼리 → 날짜 키로 병합
type DayData = {
  date: string; // 'YYYY-MM-DD'
  episodes: Episode[];
  events: ProjectEvent[]; // project_name 포함
  spending: { total: number; items: Purchase[] };
};
```
