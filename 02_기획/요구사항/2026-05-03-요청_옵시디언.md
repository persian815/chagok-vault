## 핵심 연결: CLAUDE.md가 허브

프로젝트 루트에 `CLAUDE.md` 파일 하나를 두면 Obsidian ↔ Claude Code가 자동으로 연결됩니다. Claude Code는 세션 시작 시 이 파일을 자동으로 읽습니다.

```markdown
# CLAUDE.md (프로젝트 루트)

## 현재 상태
- 단계: 기능 추가 중
- 다음 목표: 결제 모듈 구현

## 기술 스택
- Frontend: Next.js 14 + TypeScript
- Backend: FastAPI
- DB: PostgreSQL + Redis

## 핵심 설계 원칙
(Obsidian의 03_architecture.md에서 복붙)

## 브랜치 전략
- main: 프로덕션
- develop: 통합
- feature/*: 기능 개발
```

Obsidian에서 설계를 업데이트할 때마다 이 파일도 동기화해두면, Claude Code가 매번 새로 컨텍스트를 설명 안 해도 됩니다.

---

## Obsidian → Claude Code 흐름

**1. 설계 노트를 프롬프트로 변환**

`02_design.md`에 이런 섹션을 추가해두세요:

```markdown
## Claude Code 프롬프트 템플릿

### 현재 태스크
구현할 것: 사용자 인증 모듈
참고할 설계: 위의 ERD + 시퀀스 다이어그램
제약: JWT, httpOnly 쿠키 사용
```

Claude Code 세션에서 이 섹션을 그대로 붙여넣으면 컨텍스트 손실 없이 구현 시작 가능합니다.

---

## Claude Code → GitHub 흐름

Claude Code에서 바로 Git 작업을 시킬 수 있습니다:

```
# Claude Code에게 지시
"구현한 내용을 커밋하고 PR 생성해줘.
커밋 메시지는 Conventional Commits 형식으로."
```

Claude Code가 자동으로:

1. `git add` / `git commit -m "feat: add user auth module"`
2. `git push origin feature/user-auth`
3. PR 본문 초안까지 작성

---

## GitHub → Obsidian 흐름

**이슈를 백로그로 가져오기**

`07_features.md`에 GitHub 이슈 번호를 연결해두면:

```markdown
## 백로그

### 진행 중
- [ ] 결제 모듈 [#23](https://github.com/...)
  - Claude Code 프롬프트: "Stripe 연동, 03_architecture.md 참고"

### 완료
- [x] 인증 모듈 [#15](https://github.com/...)
  - 구현 노트: 04_dev_log.md#2024-01-15
```

## 개발 세션 루틴 (실전)

매 코딩 세션을 이 순서로 하면 세 도구가 자연스럽게 맞물립니다:

```
① Obsidian 열기
   → 04_dev_log.md에 오늘 날짜 섹션 추가
   → 07_features.md에서 오늘 할 이슈 확인

② Claude Code 실행
   → CLAUDE.md 자동 로딩됨
   → Obsidian 프롬프트 템플릿 붙여넣고 구현 시작

③ 구현 중 막히면
   → Claude에게 물어보고 답변을 Obsidian에 기록
   → 중요한 결정은 03_architecture.md에 ADR로 추가

④ 세션 마무리
   → Claude Code에게 커밋 + PR 요청
   → GitHub 이슈 상태 업데이트
   → Obsidian dev_log에 "오늘 한 것 / 내일 할 것" 마무리
```

## 유용한 Obsidian 플러그인 세팅

**Git 플러그인** — Obsidian 노트도 버전 관리 (프로젝트 레포와 별도 레포 권장)

**Templater** — 세션 시작 시 날짜 자동 입력:

```
## {{date:YYYY-MM-DD}} 세션

### 오늘 할 것
- [ ] 

### GitHub 이슈
- 

### Claude Code 프롬프트
\```
\```

### 막힌 것 / 해결 방법

### 내일 할 것
```

**Dataview** — 전체 프로젝트 현황 한눈에 보기:

```dataview
TASK FROM "Projects/my-webapp"
WHERE !completed
```

이 구조가 익숙해지면 "Obsidian에서 생각 → Claude Code로 실행 → GitHub에 저장" 이 루프가 거의 자동으로 돌아가게 됩니다. 특정 단계 더 자세히 파보고 싶은 부분 있으신가요?
