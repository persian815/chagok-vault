# Obsidian + Claude Code 연결 워크플로우

> 요청 날짜: 2026-05-03
> 답변 날짜: 2026-05-03

---

## 요청 요약

바이브 코딩 과정(아이디어 → 설계 → 구현 → 배포 → 기능 추가)을 Obsidian에 기록하면서 Claude Code, GitHub과 연결하는 방안을 묻는 요청.

---

## 답변

이미 구축되어 있습니다.

요청에서 제안한 구조를 현재 CHAGOK에서 거의 그대로 실현하고 있습니다.

| 요청에서 제안한 것 | 현재 CHAGOK 구조 |
|---|---|
| CLAUDE.md가 허브 역할 | `projects/littleRecords/CLAUDE.md` — 기술스택, 구조, 패턴 전체 정리 |
| Obsidian에 설계 노트 | `vault/02_프로젝트/알뜰살뜰/` — 개요, 설계, 기능목록, 트러블슈팅, 회고 |
| Claude Code → GitHub 커밋 | `/skills-push`, `/skills-cr` 스킬로 자동화 |
| 개발 세션 루틴 | `/skills-blog`, `/skills-claude-post` 로 세션 마무리 기록 |
| 요청/답변 아카이빙 | `vault/05_요청` → `vault/06_응답` 구조 (`/skills-request`) |

### 현재 구조의 흐름

```
① Obsidian (vault/)
   → 02_프로젝트/알뜰살뜰/ 에서 설계 확인 및 기록

② Claude Code (projects/littleRecords/)
   → CLAUDE.md 자동 로딩으로 컨텍스트 유지
   → .claude/commands/ 슬래시 커맨드로 반복 작업 자동화

③ GitHub (chagok-vault, littleRecords 레포)
   → 코드 이력 + 발행글 이력 분리 관리
```

### 지금 바로 활용할 수 있는 루틴

```
세션 시작
  → vault/02_프로젝트/알뜰살뜰/02_기능목록.md 열어서 오늘 할 것 확인

구현 중
  → 막히는 것은 vault/05_요청/ 에 .md로 작성 → /skills-request 실행
  → 해결된 패턴은 vault/01_공통지식/삽질모음.md 에 기록

세션 마무리
  → /skills-blog 로 개발일지 발행
  → /skills-claude-post 로 클로드 일기 발행
  → vault/02_프로젝트/알뜰살뜰/04_회고.md 주기적으로 업데이트
```
