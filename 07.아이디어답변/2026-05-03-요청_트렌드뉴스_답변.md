# 트렌드뉴스 자동화 앱 설계

> 요청 날짜: 2026-05-03
> 답변 날짜: 2026-05-03

---

## 요청 요약

보험업 트렌드 뉴스를 카테고리별(상품/마케팅/UX/기술)로 자동 수집하고, 담당자에게 이메일 발송, 월간 인사이트 공유 지원까지 하는 웹앱 자동화 설계 요청. 회사 인터넷 차단 환경이라 개인 노트북에서 실행 → 회사 이메일로 전달하는 구조 필요.

---

## 답변

### 전체 흐름

```
[오전 8시 — 자동]
  Vercel Cron → 뉴스 수집 API (Perplexity)
             → AI 카테고리 분류 (Claude API)
             → Supabase DB 저장

[오전 9시 — 자동, 평일만]
  Vercel Cron → 평일/공휴일 체크
             → DB에서 오늘 뉴스 조회
             → 담당자별 필터링
             → 이메일 발송 (Resend)

[웹 대시보드]
  브라우저 → 대시보드 / 뉴스목록 / 설정
```

---

### 기술 스택

| 구분 | 선택 | 이유 |
|---|---|---|
| Frontend | Next.js 14 + TypeScript | 알뜰살뜰과 동일, 익숙함 |
| DB | Supabase PostgreSQL | 동일 |
| 뉴스 검색 | Perplexity API | 웹 검색 + 요약 동시 처리 |
| AI 분류 | Claude API (Haiku) | 빠르고 저렴, 카테고리 분류에 적합 |
| 이메일 | Resend | API 심플, 무료 플랜 충분 |
| 스케줄러 | Vercel Cron Jobs | 별도 서버 불필요 |
| 공휴일 | 한국천문연구원 Open API | 공식 공휴일 데이터 |
| 배포 | Vercel | 동일 |

### DB 스키마

```sql
-- 뉴스 기사
news_articles (
  id, date, title, summary, category,
  source, url, ai_insight,
  assignee_opinion,  -- 담당파트 의견 (웹에서 입력)
  is_monthly_pick,   -- 월간 인사이트 공유 선정 여부
  created_at
)

-- 카테고리별 담당자
assignees (
  id, category, name, email
)
-- category: 'product' | 'marketing' | 'ux' | 'tech'

-- 이메일 발송 로그
email_logs (
  id, sent_at, recipient_email, category,
  article_count, status
)
```

```typescript
function getTargetDateRange(today: Date) {
  const day = today.getDay(); // 0=일, 1=월
  if (day === 1) {

    return { from: subDays(today, 3), to: subDays(today, 1) };
```

#### Vercel Cron 설정 (`vercel.json`)

```json
{
  "crons": [
    { "path": "/api/cron/collect", "schedule": "0 23 * * *" },
    { "path": "/api/cron/send-email", "schedule": "0 0 * * 1-5" }
  ]
}
```
> UTC 기준: 오전 8시 KST = 전날 23:00 UTC, 오전 9시 KST = 00:00 UTC

#### 월간 인사이트 공유 지원
- 뉴스목록에서 기사별 "월간 픽 선정" 토글
- 월말 알림: 마지막 주 월요일에 "이번 달 픽 선정 현황" 이메일 자동 발송
- 카테고리별 2~3건 선정 현황을 대시보드에 표시
