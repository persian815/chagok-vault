# 트렌드뉴스 자동화 앱 설계 제안

> 요청 날짜: 2026-05-04  
> 답변 날짜: 2026-05-06

---

## 요청 요약

보험업 비대면채널 트렌드 센싱을 위해 매일 ChatGPT 프롬프트로 뉴스를 수집하고 담당자에게 이메일로 전송하는 과정을 자동화하고, 대시보드·뉴스목록·설정 3개 메뉴를 갖춘 웹 앱으로 구현하는 것이 목표. 회사망 인터넷 불가 환경이라 개인 노트북에서 실행되어야 하며, Confluence 자동 등록까지 연결되어야 한다.

---

## 답변

### 전체 시스템 흐름

```
[개인 노트북]
   ↓ 08:00 스케줄러 실행
AI API (OpenAI / Claude) → 뉴스 수집 및 .md 생성
   ↓ 저장
로컬 DB (SQLite)
   ↓ 09:00 스케줄러 실행
Gmail API → 카테고리별 담당자에게 이메일 발송
   ↓
Confluence REST API → 페이지 자동 등록
```

---

### 핵심 구성 요소

#### 1. AI 뉴스 수집 모듈 (08:00 실행)

- OpenAI API 또는 Claude API를 사용해 기존 프롬프트를 자동 실행한다.
- 기존에 사용하던 `YYMM 월간보험` 트리거 방식을 데일리 방식으로 전환한다.
  - 데일리 프롬프트: "어제 날짜(YYYY-MM-DD) 기준 국내 보험시장 비대면채널 관련 뉴스를 상품/마케팅/UX/기술 카테고리로 정리해줘"
- 결과물을 `.md` 파일로 `/news/YYYY-MM-DD.md` 경로에 저장한다.

#### 2. 공휴일·주말 처리 로직

- `python-holidays` 라이브러리로 한국 공휴일을 자동 감지한다.
- 월요일에는 토요일+일요일 뉴스를 합산하여 1건으로 발송한다.
- 공휴일 다음 평일에는 공휴일 기간 뉴스를 통합 발송한다.
- 토·일·공휴일 당일은 스케줄러 자체를 실행하지 않는다.

```python
# 날짜 범위 계산 예시
def get_news_date_range(today):
    kr_holidays = holidays.Korea(years=today.year)
    target = today - timedelta(days=1)
    dates = []
    while target.weekday() >= 5 or target in kr_holidays:
        dates.insert(0, target)
        target -= timedelta(days=1)
    dates.insert(0, target)
    return dates  # 조회해야 할 날짜 목록
```

#### 3. 이메일 발송 모듈 (09:00 실행)

- Gmail API를 사용해 카테고리별 담당자에게 HTML 이메일을 발송한다.
- 담당자 매핑 구조 (설정 메뉴에서 관리):

| 카테고리 | 담당 파트 | 담당자 이메일 |
|----------|-----------|--------------|
| 상품 | 디지털사업지원 | (설정에서 입력) |
| 마케팅 | 디지털영업부 | (설정에서 입력) |
| UX | 플랫폼기획 | (설정에서 입력) |
| 기술 | 플랫폼개발 | (설정에서 입력) |

- 이메일 본문은 Markdown을 HTML로 변환하여 가독성 있게 전송한다.
- 전체 요약본과 카테고리별 개인 발송 두 가지 모드를 지원한다.

#### 4. Confluence 자동 등록 모듈

- Confluence REST API (`/rest/api/content`)를 사용한다.
- .md를 Confluence Storage Format(XML)으로 변환하여 지정된 Space에 페이지를 생성한다.
- 페이지 제목 형식: `[트렌드뉴스] YYYY-MM-DD`
- 부모 페이지(예: "트렌드뉴스 아카이브") 아래 자동으로 생성된다.

#### 5. 월간 인사이트 공유회 자동화

- 매월 마지막 주 월요일에 자동으로 트리거된다.
- 해당 월의 뉴스 중 각 카테고리별로 조회수·중요도 기반 상위 2~3건을 자동 선정한다.
- 공유회용 요약 자료를 별도 .md로 생성하고 이메일로 전체 발송한다.

### 화면 구성 (3개 메뉴)

#### 대시보드
- 이번 주 발송 현황 (카테고리별 건수)
- 오늘의 뉴스 상태 (수집 완료 / 발송 완료 / Confluence 등록 완료)
- 월간 인사이트 공유회 D-Day 표시
- 최근 7일 뉴스 수집 이력 타임라인

#### 뉴스목록
- 날짜별 뉴스 브라우징
- 카테고리 필터 (상품 / 마케팅 / UX / 기술)
- 개별 뉴스 .md 전문 보기
- Confluence 등록 상태 표시

#### 설정
- 카테고리별 담당자 이메일 등록 및 수정
- 발송 스케줄 시간 설정 (기본: 수집 08:00 / 발송 09:00)
- Gmail 계정 연동 설정
- Confluence 서버 URL / Space Key / 부모 페이지 ID 설정
- AI API Key 설정 (OpenAI 또는 Claude)
- 공휴일 목록 수동 추가 (회사 자체 휴일 등)

### 기술 스택 제안

| 영역 | 기술 |
|------|------|
| Backend | Python + FastAPI |
| Scheduler | APScheduler |
| AI API | OpenAI API 또는 Claude API |
| 이메일 | Gmail API (google-auth + googleapiclient) |
| Confluence | Confluence REST API (requests) |
| Frontend | React 또는 HTML/Tailwind CSS |
| DB | SQLite (로컬, 설정 및 이력 저장) |
| 공휴일 | python-holidays |

> 모두 개인 노트북에서 독립 실행 가능한 구성이다.

---

### 주의사항

- **Confluence 연동**: 회사 Confluence가 On-Premise인 경우 내부망에서만 접근 가능하다. 개인 노트북이 VPN 또는 사내망에 연결된 상태에서만 등록이 가능하므로, 출근 후 자동 등록되도록 Confluence 모듈만 시간을 별도로 설정하는 것을 권장한다.
- **Gmail API**: 회사 이메일이 Google Workspace인 경우 OAuth 설정이 필요하다. 일반 Gmail이면 앱 비밀번호로 SMTP 사용도 가능하다.
- **AI API 비용**: 데일리 실행 시 월 약 30회 호출이 발생한다. 프롬프트가 길므로 Claude API의 경우 프롬프트 캐싱 기능을 활용하면 비용을 절감할 수 있다.

5. **웹 UI 구성** — 대시보드 → 뉴스목록 → 설정 순서로 구현 (3~5일)
6. **월간 공유회 모듈 추가** — 마지막 주 자동 트리거 및 요약 자료 생성 (1~2일)
